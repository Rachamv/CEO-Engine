📐 CEO Method Terms
Order Block (OB)
The last opposite-direction candle right before a strong impulsive move. Price often returns to this zone once ("mitigation") before continuing in the original direction — that return is a common entry area.
Fair Value Gap (FVG)
A 3-candle imbalance: the gap left between candle 1's high/low and candle 3's low/high when the middle candle moves fast enough to skip a price range. Price tends to revisit ("fill") this gap later.
Quasimodo (QM)
A failed order block — structure breaks one way, then sharply reverses through it. The CEO method treats this as its highest-probability entry, since it traps traders positioned on the wrong side first.
Break of Structure (BOS)
Price closes beyond a prior swing high/low, confirming momentum in that direction. A "Double BOS" is two consecutive breaks — a stronger, more protected structural signal, shown on the chart as a bigger arrow marker.
Fibonacci 50% (Equilibrium)
The midpoint of the most recent swing. CEO entries prefer the "discount" zone (below 50% in an uptrend) or "premium" zone (above 50% in a downtrend) — buying/selling at a relative bargain within the current leg.
Confluence
How many independent signal models agree on the same trade at the same time. Higher confluence counts mean more independent confirmation, not just one method's opinion.
Regime
The engine's read on current market character — trending up, trending down, ranging, or choppy. Some signal types are only allowed to fire in regimes they're suited for.
Session
The trading window a bar falls in: London, New York, or Asian. The London/New York overlap is the highest-liquidity period for Forex and Gold, which is why it's checked by default.
CEO Valid
The full structure sequence has aligned on one bar — BOS, then OB/QM, then the Fib discount/premium zone — which is the actual entry trigger. This is the "CEO" marker on the chart.
⚙️ Settings Explained
Risk per trade (%)
The percentage of your account balance you're willing to lose if a trade's stop-loss is hit. Position size (lots) is calculated backward from this and your SL distance — you never manually set lot size.
Min signal quality
A 0–100 score every signal must clear before it's considered at all. Raising it means fewer, more selective trades; lowering it means more signals but more noise.
Min model consistency
Filters out signal models whose historical win rate hasn't held up consistently across walk-forward validation windows. 0% disables this filter entirely.
Confluence mode
Controls what the "Confluence" signal type requires: Sweep (filter-models agree), CEO structure (full structural sequence required), or Full (both — strictest).
Daily Loss Limit / Max Drawdown
Prop-firm style risk rules. Once either is breached, the funded account guard halts new trades for the rest of the day (or entirely, for max drawdown) — check your firm's actual rules and match them here.
Auto-trade
When on, the engine places real orders on MT5 automatically whenever a signal fires. When off, you get alerts (console/Telegram/dashboard) only — you place trades yourself.
MTF mode (bias / confirm / score)
How multiple timeframes must agree before a multi-timeframe signal counts: Bias (majority agree), Confirm (all must agree — strictest), or Score (weighted alignment across all selected TFs).
Performance Feedback
Automatically reduces risk or pauses the engine after a losing streak or a falling rolling win rate, so a rough patch doesn't compound. Configurable rolling window, loss-streak threshold, and win-rate floor.
Walk-forward validation
Splits historical data into sequential windows and re-tests on each out-of-sample, so a strategy has to prove itself repeatedly rather than just fitting one lucky stretch of history.
News gate
Blocks new signals around high-impact scheduled economic releases (configurable minutes before/after), since volatility around news often invalidates normal structure.
Dashboard port
The local port this web dashboard runs on (default 5000). Only change it if that port's already in use on your machine.
🛠️ Troubleshooting
MT5 won't connect / "not running or not logged in"
Open the actual MetaTrader 5 desktop terminal first and log into your broker there — this dashboard connects to that running terminal, it doesn't log in independently. Then retry Detect in Setup.
Telegram alerts not arriving
Use the "Test" button in Settings to check your token/chat ID immediately. If "Find Chat ID" returns nothing, send your bot any message first — Telegram only reports chats it has heard from.
Dashboard requires saved settings
Complete and save the Setup wizard (or Settings panel) at least once before using the dashboard. It reads its configuration from that saved file.
Chart looks empty
Live market data appears once MT5 is connected and the dashboard has received at least one recent price update for that symbol/timeframe.
Dashboard password
Auto-generated and printed once to your terminal/console when the engine starts. It's also saved to ceo_dashboard_auth.json in the project folder — check there if you missed the console output.
